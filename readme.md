## Тестовое задание "Баннер для мобильного приложения"

- Реализация адаптива: 

    - изначально стилизую баннер по макету для iphone 12 (ширина экрана 390px), 

    - затем при помощи media-запросов делаю адаптивную вёрстку для iphone SE, 8+ (отдельно стилизую уменьшение карточек для ширины экрана не более 375px, и отдельно - уменьшение отступов для высоты не более 736px), также для высоты не более 736px поднимаю фоновое изображение на 32px (в файле reboot.css) 

    - и при помощи media-запросов делаю адаптивную вёрстку для iphone 8 (ширина экрана не более 320px), при этом уменьшаю размер некоторых шрифтов, отступов и элементов, чтобы контент поместился на экране.

    - если ширина экрана не менее 414px, при помощи media-запроса увеличиваю ширину контейнера.

- Изображения: использую отдельные версии для iphone 12 и для iphone SE, 8+. А также версии @2x, @3x, @4x для ретинизации.

- Логика: кнопки выбора способа оплаты (WEEKLY или YEALY) является input-ом с типом radio. При изменении состояния (то есть при выборе определенного radio) срабатывает inline-обработчик события onchange, который меняет атрибут href у ссылки continue. 

- Перевод на другие языки: выполняется в скрипте script.js.

- При переводе на французский добавляю класс для уменьшения размера главного заголовка, чтобы он поместился на экране.

- Адаптация (ограничение ширины контента) на кнопках выбора способа оплаты под другие языки: выполнено средствами css, при помощи width, max-width и подобных методов: на языках, на которых ширина блока слишком велика, срабатывает max-width для красивого размещения контента.
